#### 详细协作步骤

**Step 1: RouterAgent (路由Agent)** - [router.py](file:///Users/jinniu/Documents/GitHub/ClinicVoxAI/app/agents/router.py)

```python
def handle(self, state: SessionState, text: str) -> SessionState:
    # 1. 分类意图和患者类型
    patient_type, intent, confidence = self.classify(text)
    
    # 2. 意图延续处理 (处理follow-up情况)
    if intent == "other" and state.intent in {"schedule", "report", ...}:
        intent = state.intent  # 延续之前的意图
    
    # 3. 更新状态
    state.patient_type = patient_type
    state.intent = intent
    state.intent_confidence = confidence
```

**Step 2: SupervisorAgent (监督Agent)** - [supervisor.py](file:///Users/jinniu/Documents/GitHub/ClinicVoxAI/app/agents/supervisor.py)

```python
def decide(self, state: SessionState) -> str:
    # 低置信度 → 需要澄清
    if state.intent_confidence < 0.5:
        return "clarify"
    
    # 错误过多 → 转人工
    if len(state.errors) >= 2:
        return "handoff"
    
    # 正常流程 → 继续执行
    return "proceed"
```

**Step 3: TaskAgent (任务Agent)** - [task.py](file:///Users/jinniu/Documents/GitHub/ClinicVoxAI/app/agents/task.py)

根据意图执行不同任务：
- `registration/schedule`: 患者注册 + 预约
- `reschedule/cancel`: 修改/取消预约
- `report`: 获取医疗报告摘要

### 总结

这是一个基于 **LangGraph** 的多Agent协作语音AI诊所助手系统：

1. **入口层**: FastAPI 提供 REST API 和 WebSocket 接口
2. **编排层**: Orchestrator + LangGraphRuntime 管理整体流程
3. **图执行层**: LangGraph 定义 Agent 节点和条件路由
4. **Agent层**: Router → Supervisor → Task 的协作链

6. **语音层**: 支持多种语音流接入方式

### 二、LangGraph 状态图结构

核心协作通过 **LangGraph 状态图** 实现，定义在 [langgraph_flow.py](file:///Users/jinniu/Documents/GitHub/ClinicVoxAI/app/graph/langgraph_flow.py)：

```python
graph = StateGraph(SessionState)

# 添加三个核心节点
graph.add_node("router", router_node)      # RouterAgent
graph.add_node("supervisor", supervisor_node)  # SupervisorAgent
graph.add_node("task", task_node)          # TaskAgent

# 设置执行流程
graph.set_entry_point("router")            # 入口：Router
graph.add_edge("router", "supervisor")     # Router → Supervisor (固定边)
graph.add_conditional_edges("supervisor", route_decision)  # Supervisor → 条件路由
graph.add_edge("task", "__end__")          # Task → 结束
```

### 六、条件路由逻辑详解

[route_decision](file:///Users/jinniu/Documents/GitHub/ClinicVoxAI/app/graph/langgraph_flow.py#L70-L80) 函数实现智能路由：

```python
def route_decision(state: SessionState | dict) -> str:
    decision = state.meta.get("decision") or self.supervisor.decide(state)
    
    if decision == "handoff":
        return "__end__"  # 直接结束，转人工
    
    if decision == "clarify":
        count = state.meta.get("clarify_count", 0) + 1
        state.meta["clarify_count"] = count
        if count >= 2:  # 最多澄清 2 次
            return "__end__"
        return "router"  # 返回重新分类
    
    return "task"  # 正常执行任务
```

**路由决策树**：

```
                    Supervisor.decide()
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    confidence<0.5    errors>=2      其他情况
          │               │               │
          ▼               ▼               ▼
       clarify         handoff         proceed
          │               │               │
          │               │               │
     ┌────┴────┐          │               │
     │         │          │               │
     ▼         ▼          ▼               ▼
clarify_count  ≥2?    结束           执行 Task
     │         │
     │         └──────► 结束
     │
     └─────► 返回 Router
```

### 七、辅助智能体的使用

**ContextAgent** 和 **AuditAgent** 虽然不在 LangGraph 主流程中，但可以在特定场景下使用：

```python
# ContextAgent - 生成上下文摘要
class ContextAgent:
    def handle(self, state: SessionState, text: str):
        summary = {
            "intent": state.intent,
            "patient_type": state.patient_type,
            "verified": state.verification_status == "verified",
        }
        state.meta["context_summary"] = summary
        return state

# AuditAgent - 审计追踪
class AuditAgent:
    def handle(self, state: SessionState, text: str):
        state.meta["audit_note"] = "audit_branch"
        return state
```

它们可以在以下场景使用：
- 在 Supervisor 决策后调用 ContextAgent 生成摘要
- 在流程结束时调用 AuditAgent 记录审计信息
- 作为独立的工具节点在需要时插入流程

### 八、关键设计模式

1. **责任链模式**：Router → Supervisor → Task 形成处理链
2. **状态模式**：SessionState 在节点间传递，携带所有上下文
3. **策略模式**：SupervisorPolicy 可配置决策策略
4. **工厂模式**：`build_agents()` 根据配置创建不同后端的 Agent

### 总结

这是一个典型的 **LangGraph 多 Agent 协作系统**：

- **3 个核心 Agent** 通过状态图紧密协作
- **SessionState** 作为共享状态在节点间传递
- **Supervisor** 作为决策中心控制流程走向
- **条件路由** 实现智能的流程控制（澄清/转人工/执行）
- **2 个辅助 Agent** 提供额外功能（上下文/审计）

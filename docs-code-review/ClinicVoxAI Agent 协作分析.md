
          
# ClinicVoxAI Agent 协作分析

## Agent 数量

ClinicVoxAI 系统中共有 **5 个 Agent**，它们分别是：

1. **RouterAgent** - 负责意图识别和任务路由
2. **SupervisorAgent** - 监督和协调其他 Agent，验证置信度和处理错误
3. **TaskAgent** - 处理具体业务任务，执行工具操作
4. **ContextAgent** - 管理会话上下文和状态
5. **AuditAgent** - 负责审计和合规检查

### 详细协作流程

1. **初始化阶段**：
   - 系统启动时，`build_agents()` 函数创建所有 Agent 实例
   - `LangGraphRuntime` 构建包含 Router、Supervisor 和 Task Agent 的工作流

2. **请求处理流程**：
   - **Step 1**: Router Agent 读取用户输入文本，识别意图并设置置信度
   - **Step 2**: Supervisor Agent 验证意图置信度，检查是否存在错误
   - **Step 3**: Supervisor 做出决策：
     - 如果置信度低或需要澄清，返回 "clarify" 决策
     - 如果需要人工干预，返回 "handoff" 决策
     - 否则，继续执行任务
   - **Step 4**: Task Agent 执行具体任务：
     - 从文本中提取所需字段
     - 如果字段充足，调用相应工具执行操作
     - 生成响应并更新状态
   - **Step 5**: 系统返回响应给用户

3. **错误处理与异常流程**：
   - 工具错误 → 添加到 state.errors → Supervisor 决定是澄清还是转交
   - 低置信度 → 循环回到 Router 进行澄清
   - 多次澄清后仍无法理解 → 结束对话或转交人工

4. **意图切换流程**：
   - 如果用户在通话中途改变意图，Router 会更新意图并重置上下文
   - Task Agent 根据新意图切换工作流程

5. **状态管理**：
   - 所有 Agent 通过共享的 SessionState 进行协作
   - 状态变更会被记录和追踪
   - Context Agent 负责维护和更新会话上下文

### 技术实现

- **工作流引擎**：基于 LangGraph 构建的状态驱动工作流
- **状态传递**：Agent 之间通过 SessionState 共享信息，而非直接调用
- **条件分支**：Supervisor 的决策决定工作流的下一步走向
- **工具集成**：Task Agent 集成了多种工具（数据库、日历、报告等）

这种基于 LangGraph 的 Agent 协作架构提供了灵活、可扩展的对话处理能力，能够有效应对医疗场景中的各种复杂交互需求。

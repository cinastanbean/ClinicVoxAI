
Product Requirements Document - PRD 风格


### 项目名称
语音智能诊所客服系统（Voice AI Clinic Assistant）

### 项目目标
开发一个基于电话的AI语音客服系统，针对美国小型诊所（尤其是独立诊所、专科诊所），处理患者来电的主要需求。系统需支持自然语音交互、多Agent协作、HIPAA合规，并能无缝转接人工。目标是减少前台人工负担、提升患者体验、处理高峰期呼叫。

### 核心用户场景与功能需求
患者通过普通电话拨打诊所号码，系统自动接听并进行语音交互。主要分为两大类患者：

1. **新患者（首次就诊）**
   - 引导完成基本注册：收集姓名、出生日期、联系电话、邮箱、保险信息（基本字段，如保险提供商、会员ID）、主要症状/就诊原因。
   - 隐私声明：开头播放HIPAA隐私提醒并获取口头同意。
   - 预约引导：根据症状初步分诊（简单意图识别），推荐合适医生/科室，并查询可用时间槽进行预约。
   - 结束时发送确认短信/邮件（包含预约详情、诊所地址等）。

2. **老患者（已有病例记录）**
   - 身份验证：通过手机号、姓名+出生日期、或上次就诊ID进行验证（支持语音模糊匹配）。
   - 支持的主要意图（多轮对话，支持意图切换）：
     - 预约/改约/取消就诊：查询医生日历、展示可用时间段（语音播报选项，如“周三上午10点或下午2点”），患者语音选择并确认。
     - 查询/解释检查报告：检索历史报告（实验室结果、影像报告等），用通俗语言总结关键发现、医生建议；如患者追问细节，可提供更深入解释（避免诊断性语言，只转述医生原意）。
     - 其他常见咨询：用药提醒、复诊建议、保险覆盖确认、症状初步咨询（引导至合适医生或转人工）。
   - 如果意图超出范围或复杂（如需详细病史讨论），自动转人工座席。

### 系统架构与技术要求（多Agent协作）
- **整体架构**：多Agent系统（推荐使用LangGraph、CrewAI或AutoGen框架）。
  - 路由Agent（Router Agent）：首轮对话分析意图，判断新/老患者，路由到对应Agent。
  - 注册Agent：处理新患者信息收集、存储到数据库。
  - 身份验证Agent：老患者验证逻辑。
  - 预约Agent：集成日历系统（Google Calendar / Microsoft Bookings / 或自定义），管理时间槽、冲突检测、确认/取消。
  - 报告Agent：查询数据库，提供报告摘要和自然语言解释（使用LLM fine-tune医疗术语简化）。
  - 通用对话Agent：处理闲聊、澄清、重听等。
- Agent间协作：共享上下文（病人ID、会话历史），支持工具调用（查数据库、查日历、发短信）。
- 异常处理：语音识别失败时重问；沉默/挂断处理；最大对话轮次限制。

### 输入/输出与技术栈建议
- **电话接入**：Twilio（入站呼叫、实时STT/TTS）或Retell AI（内置语音AI支持）。
- **语音处理**：
  - STT：Deepgram / AssemblyAI / Google Cloud Speech-to-Text（高准确率，支持医疗词汇）。
  - TTS：ElevenLabs / Amazon Polly / PlayHT（自然美式英语声音，支持情感表达）。
- **LLM核心**：OpenAI GPT-4o / Grok / Claude（支持function calling和多Agent）。
- **数据库**：HIPAA合规选项，如AWS RDS (with BAA)、Google Cloud SQL、或Supabase/PostgreSQL + 加密。
- **后端**：Python (FastAPI) + LangChain/LangGraph。
- **合规与安全**：
  - 全程加密传输。
  - 不存储原始语音录音（只存文本日志）。
  - 签署BAA（Business Associate Agreement）。
  - 明确AI非诊断工具，转人工机制。
  - 所有PHI（受保护健康信息）加密存储。

### 非功能需求
- 支持英语（美式口音优先），未来可扩展西班牙语。
- 响应延迟：< 2秒（端到端）。
- 日呼叫容量：初始支持50-100并发（可扩展）。
- 监控与日志：通话成功率、意图识别准确率、转人工率。
- 成本控制：优先使用高效模型，避免过度调用。

### 成功标准（MVP）
- 80%+ 的简单预约/报告查询能全自动完成，无需转人工。
- 患者满意度（通话后短信调研）> 4.0/5。
- HIPAA合规通过基本审计。

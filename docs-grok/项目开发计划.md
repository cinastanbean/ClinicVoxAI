### 项目开发计划：语音智能诊所客服系统（Voice AI Clinic Assistant）

以下是基于您提供的项目需求生成的完整开发计划。我将按要求逐一展开，包括技术架构图（以文字描述形式呈现，便于理解和复制）、MVP开发Roadmap、核心代码结构建议、潜在风险与Mitigation，以及初步Prompt工程示例。计划假设使用Python + LangGraph作为多Agent框架，结合Twilio作为电话接入点，OpenAI GPT-4o作为LLM核心，以确保高效开发。整个计划针对MVP版本，聚焦核心功能，目标是快速迭代并验证成功标准。

#### 1. 技术架构图（文字描述）
架构采用多Agent协作模式，使用LangGraph构建状态图（Graph），每个Agent作为节点，共享上下文（State）。整体流程：电话呼入 → 语音处理 → Router Agent路由 → 特定Agent处理 → 输出语音/动作。以下是文字描述的架构图（使用ASCII艺术表示层次结构）：

```
[电话接入层] (Twilio Webhook)
  |
  v
[语音处理层] (STT: Deepgram → TTS: ElevenLabs)
  |  (实时转录/合成)
  v
[AI核心层] (LangGraph State Graph)
  - State: {patient_id, session_history, intent, verified, data_collected}
  |
  +-- Router Agent (意图识别: GPT-4o prompt)
      |  (判断新/老患者，路由)
      +-- 新患者路径 --> Register Agent (收集信息 → DB存储)
      |                   + Tool: DB Insert (AWS RDS)
      +-- 老患者路径 --> Auth Agent (验证: 语音匹配 → DB查询)
                          + Tool: DB Query (patient verification)
                          |
                          +-- 成功 --> Intent Handler (分支)
                               +-- Appointment Agent (预约/改约/取消)
                               |   + Tools: Calendar API (Google Calendar), SMS/Email (Twilio)
                               +-- Report Agent (报告查询/解释)
                               |   + Tools: DB Query (reports), LLM Summarize
                               +-- General Agent (其他咨询/闲聊/澄清)
                                   + Tools: Fallback to human, Clarify prompt
  |
  v
[后端服务层] (FastAPI Server)
  - Endpoints: /twilio_webhook (handle calls), /callback (async tools)
  - Database: AWS RDS (HIPAA-compliant, encrypted PHI)
  - Security: HTTPS, BAA with vendors
  |
  v
[监控层] (Logs: ELK Stack or Sentry; Metrics: Prometheus)
  - Track: Call success, intent accuracy, transfer rate
```

此架构确保Agent间协作通过共享State实现，例如Router Agent设置intent后，Appointment Agent可直接访问session_history。异常（如STT失败）通过Graph的条件边（conditional edges）重路由到澄清节点。

#### 3. 核心代码结构建议（Python + LangGraph）
使用LangGraph构建多Agent系统，代码结构采用模块化设计，便于扩展。根目录结构如下：

```
project/
├── app.py               # FastAPI主应用，处理Twilio Webhook
├── agents/              # Agent定义
│   ├── router.py        # Router Agent
│   ├── register.py      # Register Agent
│   ├── auth.py          # Auth Agent
│   ├── appointment.py   # Appointment Agent
│   ├── report.py        # Report Agent
│   └── general.py       # General Agent
├── tools/               # 工具函数（LangChain Tools）
│   ├── db_tools.py      # DB查询/插入（SQLAlchemy）
│   ├── calendar_tools.py# Google Calendar集成
│   └── comm_tools.py    # SMS/Email (Twilio)
├── graph.py             # LangGraph StateGraph定义
├── schemas/             # Pydantic模型（Patient, Appointment等）
├── config.py            # Env vars (API keys, DB URL)
├── prompts.py           # Prompt模板
├── utils/               # 辅助函数（STT/TTS wrapper, logging）
├── tests/               # 单元/集成测试
└── requirements.txt     # Dependencies: langgraph, langchain_openai, fastapi, twilio, sqlalchemy, etc.
```

**核心代码伪代码示例（graph.py）**：
```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
from langchain_openai import ChatOpenAI
from agents import *  # 导入所有Agent
from tools import *   # 导入工具

llm = ChatOpenAI(model="gpt-4o")

class State(TypedDict):
    patient_id: str
    session_history: list[str]
    intent: str
    verified: bool
    data_collected: dict

# 定义Graph
graph = StateGraph(State)

# 添加节点（Agents作为函数）
graph.add_node("router", router_agent)  # 返回{"intent": "appointment"}等
graph.add_node("register", register_agent)
graph.add_node("auth", auth_agent)
graph.add_node("appointment", appointment_agent.bind_tools([calendar_tool, sms_tool]))
graph.add_node("report", report_agent.bind_tools([db_query_tool]))
graph.add_node("general", general_agent)

# 条件边（基于State）
graph.add_conditional_edges("router", route_to_agent, {
    "new_patient": "register",
    "old_patient": "auth",
    "fallback": "general"
})
graph.add_edge("register", "appointment")  # 示例路径
graph.add_conditional_edges("auth", auth_result, {"success": "router", "fail": END})  # 验证失败结束
# ... 添加更多边

# 入口/出口
graph.set_entry_point("router")
graph.set_finish_point(END)  # 或转人工节点

# 编译Graph
app = graph.compile()

# 在FastAPI中使用：async def handle_call(input: str) -> str:
#     state = app.invoke({"session_history": [input]})
#     return state["output"]  # TTS输出
```

每个Agent文件（如router.py）定义为函数：`def router_agent(state: State) -> State`，内部使用LLM.invoke(Prompt + state["session_history"])解析意图。工具使用LangChain的.bind_tools()绑定。

#### 4. 潜在风险与 Mitigation
| 风险 | 描述 | 概率/影响 | Mitigation |
|------|------|-----------|------------|
| **HIPAA合规失败** | 数据泄露或未加密PHI导致罚款。 | 高/高 | - 使用AWS RDS with BAA，加密所有PHI。<br>- 定期审计日志，不存语音录音。<br>- 集成转人工机制，避免AI诊断。 |
| **语音识别准确率低** | 医疗词汇/口音导致误解。 | 中/高 | - 选择Deepgram（医疗训练模型）。<br>- 添加重问逻辑（max 3次），fallback到人工。<br>- 测试多样口音数据集。 |
| **延迟超标** | LLM/工具调用导致>2s响应。 | 中/中 | - 使用高效模型（GPT-4o mini for simple tasks）。<br>- 异步工具调用，缓存常见查询。<br>- 监控并优化（e.g., reduce prompt size）。 |
| **意图识别错误** | 多意图切换或模糊输入导致路由错。 | 高/中 | - Fine-tune Prompts with examples。<br>- 支持意图切换（Graph循环边）。<br>- 监控转人工率，迭代基于日志。 |
| **集成失败** | Twilio/Google Calendar兼容问题。 | 低/高 | - 早期原型测试。<br>- 备用方案（e.g., Microsoft Bookings）。<br>- 文档化API错误处理。 |
| **成本超支** | 高呼叫量导致LLM/Twilio费用高。 | 中/低 | - 限流（初始50并发）。<br>- 使用缓存/批处理。<br>- 监控仪表盘，警报阈值。 |
| **患者不满** | AI不自然或转人工频繁。 | 中/中 | - TTS用情感声音。<br>- Post-call调研迭代。<br>- 目标80%自动化率。 |

风险管理：每周review，优先高影响项。总风险预算：分配10%时间用于contingency。

#### 5. 初步提示工程（Prompt）示例

**意图识别Prompt（用于Router Agent）**：
```

Extract main intent: appointment, report query, registration, or other. Output JSON: {"patient_type": "new" or "old", "intent": "appointment" or "report" or "other"}. Be concise, no extra text.

Examples:
User: Hi, I'd like to book an appointment for the first time.
Output: {"patient_type": "new", "intent": "appointment"}

User: I need to check my last lab results, my name is John Doe, DOB 01/01/1980.
Output: {"patient_type": "old", "intent": "report"}

User: {user_input}
Output:
```

**报告解释Prompt（用于Report Agent）**：
```
System: You are a helpful AI summarizing medical reports in simple, non-diagnostic language. Only restate doctor's notes, do not give advice or diagnose. Use friendly tone. If details are asked, provide more from the report. End with: "For questions, consult your doctor or transfer to human."

Report Data: {report_text} (e.g., "Lab results: Hemoglobin 14g/dL (normal). Doctor note: No action needed, follow up in 6 months.")

User: Explain my blood test results.
Output: Your hemoglobin level is 14g/dL, which is within normal range according to the report. The doctor suggests no immediate action and a follow-up in 6 months. For questions, consult your doctor or transfer to human.

User: {user_input}
Output:
```

这些Prompt可通过few-shot examples fine-tune，提高准确率。建议使用LangChain的PromptTemplate管理。
